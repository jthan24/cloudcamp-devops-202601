# Contribution Guidelines

Thank you for your interest in contributing to this project! We appreciate your help and want to make contributing as smooth as possible.

## How to Contribute
1. **Fork the Repository**: Click the fork button at the top right corner of the repository to create your own copy of the project.
2. **Clone Your Fork**: Use the following command to clone your fork to your local machine:
   ```bash
   git clone https://github.com/<your-username>/cloudcamp-devops-202601.git
   ```
3. **Create a New Branch**: It’s a good practice to create a new branch for your feature or fix.
   ```bash
   git checkout -b my-feature-branch
   ```
4. **Make Your Changes**: Edit the files as needed. Make sure to follow the coding style and conventions used in the project.
5. **Commit Your Changes**: Write a clear commit message explaining your changes.
   ```bash
   git commit -m "Add new feature"
   ```
6. **Push Your Changes**: Push your changes to your fork.
   ```bash
   git push origin my-feature-branch
   ```
7. **Open a Pull Request**: Go to the original repository and click on “Pull Requests”. Click “New Pull Request” and select your branch.

## Pull Request Rules
- **Descriptive Title**: Provide a clear title that reflects your changes.
- **Detailed Description**: Explain what changes you made and why. You can reference issues by using `#issue_number`.
- **No Merge Conflicts**: Ensure that your branch is up-to-date with the base branch to avoid conflicts.
- **Follow the Code Standards**: Ensure that your code adheres to the project's coding standards.

Thank you for contributing!
